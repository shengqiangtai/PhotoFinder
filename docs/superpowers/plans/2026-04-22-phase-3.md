# Phase 3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add model download, embedding, and indexing so newly added photos can produce vectors and thumbnails.

**Architecture:** Keep Phase 3 incremental. Add a lightweight downloader that manages model file presence and progress, a singleton embedder that loads local ONNX sessions plus tokenizer, and an indexer that coordinates scan, vector generation, thumbnail generation, and database writes behind one global progress state. Update existing library/system routes to trigger indexing and expose model/index status without changing the Phase 2 API shape more than necessary.

**Tech Stack:** FastAPI, aiosqlite, sqlite-vec, onnxruntime, Pillow, httpx, qrcode, unittest, unittest.mock.
